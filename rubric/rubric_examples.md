# Rubric Examples — 8 Worked Cases

These are calibration examples drawn from the actual `Ventures_Tracker.xlsx`
existing entries. Each shows: input → reasoning → both Scouter row + Analyzed Startups row.

Use these as pattern-match references when scouting new startups.

## Example 1 — `✅ STRONG`, single-BU pilot-ready

**Input**: Xterra Robotics, IIT Kanpur incubated quadruped robot maker

**Reasoning**:
- Identity verified ✓ (CIN, MCA, IIT-K SIIC portfolio)
- Real startup ✓
- Credibility: clean — IIT-K SIIC, named founders, real product (SVAN M2)
- BU fit: HZL underground mines (4 mines, all underground, stair/uneven terrain) + IOB Karnataka rough-terrain. Other BUs ⚠️ Possible only.
- Cross-BU: NO — too HZL/IOB-specific to merit Group procurement
- CSR/ESG: NO

### Scouter row (T = Verdict cell)
```
✅ STRONG | Cross-BU — | CSR/ESG —
Indigenous quadruped maker explicitly built for stairs/uneven terrain wheeled
robots can't handle. Direct fit HZL underground mines + IOB rough-terrain.
Bootstrapped, IIT-K incubated. Pilot candidate. Founder call recommended.
```

### Analyzed Startups row
| Col | Value |
|---|---|
| B (Tier) | `P1 – Immediate` |
| C (Company) | `Xterra Robotics` |
| D (BU) | `HZL / IOB` |
| F (Action) | `Pilot at Rampura Agucha for confined-space inspection. Validate stair-climbing capability with on-site demo before scaling.` |
| G (Owner) | `HZL Mines` |
| H (Rationale) | `Indigenous quadruped, IIT-K incubated, GISTM-aligned. Bootstrapped → flexible commercial. Cobot C1 + SVAN M2 directly applicable.` |

## Example 7 — `🔍 Insufficient Data` (correct discipline)

**Input**: User asks to scout "Zayraiq Technology Solutions" — no web footprint

**Reasoning**:
- Web search returns zero hits (close variants surface: Zaytrics, ZariyahTech, Zariya, Zaytun — none confirmed)
- Cannot verify identity, founder, product
- Do NOT invent. Do NOT scout the wrong company.

### Action

Stop. Output a 🔍 row in the Scouter sheet, do NOT add to Analyzed Startups, and ask user:

```
No web footprint found under exact spelling "Zayraiq Technology Solutions".
Close variants: Zaytrics, ZariyahTech, Zariya, Zaytun, Zreyas — but none
confirmed.

Can you share:
- A website or LinkedIn link
- A pitch deck
- Founder name
- Or confirm the correct spelling?

Without one of these I'd risk scouting the wrong company under a wrong identity.
```

Add row to Scouter only:
```
B: [next #] | C: Zayraiq Technology Solutions | D: 🔍 Insufficient Data | ...
T: 🔍 INSUFFICIENT DATA | Cross-BU — | CSR/ESG —
No web footprint found under exact spelling. User asked for clarification — pending deck/website/founder name.
```

## Pattern summary

| Verdict shape | Tier | Owner | Action verb |
|---|---|---|---|
| ✅ STRONG, named BU | P1 – Immediate | BU name | "Pilot at [site]" |
| ✅ STRONG + ESG strategic | P1 – Immediate (Strategic) | Group Sustainability | "Strategic [theme] play — pilot at [site]" |
| ⚠️ + 1 BU | P2 – BU-specific | BU name | "Conduct small pilot at [site] (constraint)" |
| ⚠️ + conditions | P2 – Conditional BU Evaluation | BU name | "Evaluate selectively for [BU/use case]" |
| ⚠️ + Cross-BU | P2 – Platform / Enabler (Conditional) | Group function | "Evaluate at Group level only if [condition]" |
| ⚠️ + procurement-led | P3 – Conditional / Procurement-led | Group Procurement | "Park for Group Procurement [year]" |
| ❌ + CSR/ESG ✓ | Deprioritise (CSR / Central only) | Foundation/Group | "Route to [target]" |
| ❌ no routing | Deprioritise | – | "Skip — [reason]" |
| 🚩 Not a Startup | Deprioritise | – | "Skip — not a startup" |
| 🔍 Insufficient Data | (do NOT add to Analyzed) | – | (ask user) |
